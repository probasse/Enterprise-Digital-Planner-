Project Planner User Guide

Everything you need to plan, execute, and review a project cut-over — tasks, risks, decisions, issues, communications, rollback, and reporting in one browser-based tool.

2Getting Started

Opening the tool

Open index.html in any modern browser (Chrome, Edge, Firefox, Safari). No installation is needed.

First-time setup

A blank project called My Project is created automatically on first launch.

Recommended first steps:

  1. Click ⚙️ Settings (top-right of the header) to set your project name, cut-over date, and timezone.
  2. Go to Resources and add your team members — this lets you assign owners when creating tasks.
  3. Go to Tasks and start building your task list.

3Project Management

Project Planner supports multiple independent projects. Each project has its own tasks, resources, risks, issues, decisions, actions, communications, rollback plan, go/no-go criteria, and settings. The theme and sidebar state are shared globally.

Switching projects

Click the project name button in the header to open the switcher dropdown. Click any project name to switch immediately — no page refresh needed.

Creating a new project

  1. Click the project name button in the header.
  2. Click + New Project.
  3. Enter a project name (required), description (optional), and cut-over date (optional).
  4. Click Save New Project.

New projects start blank with default categories and statuses seeded automatically.

Deleting a project

Open ⚙️ Settings and click Delete Project in the footer. Confirm to permanently delete the project and all its data. You cannot delete the only remaining project — create a replacement first.

You can also delete a project from the switcher dropdown by clicking the ✕ next to its name.

Legacy data migration

If you used the tool before multi-project support was added, your existing data is automatically migrated to a project called Migrated Project on first launch of the updated version.

The Tasks view is the central planning register. Each task has planned and actual dates, dependencies, status history, and more.

MilestoneOptionalMarks the task as a key milestone (shown with 🔹)
Task ID (T-001, T-002, …) is assigned automatically on save and cannot be changed.

Actual dates — auto-populated

Status transitionWhat happens automatically
→ In ProgressActual Start is set to the current date/time. Only recorded on the first transition — never overwritten.
→ CompletedActual End is set to the current date/time.

Actual Start and Actual End appear as read-only fields in the task modal as soon as they have a value. They also update immediately in the task list row — no page refresh needed.

Status history

Every status change is recorded with a timestamp. Open any task to see the full history panel at the bottom of the modal — each status, when it was entered, and how long the task spent in that state.

Dependency management

Snap to Dependency: Select one or more predecessor tasks in the Depends On field, then click the ⛓ Snap to Dependency button. The start date is set to 1 second after the latest predecessor's end date, and the end date is recalculated if a duration is set.

Recalculate Dates: The Recalculate Dates button in the Tasks toolbar re-snaps the start dates of all dependent tasks in topological order — useful after rescheduling predecessors.

Column visibility

Click ⊞ Columns to show or hide columns. Preferences are saved per project. Available columns:

Task ID · Status · Task Name · Category · Priority · Assignee · Start Date · End Date · Duration · Dependencies · Actual Start · Actual End

Filtering

Use the filter bar to narrow the task list by Category, Status, Priority, and Assignee.

Quick status update

Change a task's status directly from the task list using the inline status dropdown — no modal needed. Actual dates update immediately in the row and in any open modal for that task.

6Timeline

The Timeline view renders all tasks as a Gantt-style chart.

  • Tasks appear as horizontal bars colour-coded by status.
  • Milestones appear as diamond markers.
  • Use the − and + zoom buttons to compress or expand the time axis.
  • Dependencies are shown visually between task bars.

Active vs. inactive resources

Setting a resource to Inactive removes them from all assignee dropdowns, preventing new assignments. Existing assignments are preserved and continue to display the resource's name. To re-enable, edit the resource and set Status back to Active.

9Rollback Plan

The Rollback Plan documents what to do if the cut-over needs to be reversed.

Configuration

  • Rollback Trigger Criteria — describe the conditions that would initiate a rollback (e.g. system unavailable, critical errors exceeding threshold).
  • Point of No Return — the date and time after which rollback is no longer feasible. Displayed prominently in the Rollback Plan report.

Rollback steps

10Go/No-Go

The Go/No-Go view manages the readiness checklist used to make the final decision to proceed (or not) with the cut-over.

Categories

CategoryPurpose
Technical ReadinessSystems, infrastructure, integrations
Business ReadinessProcess, data, sign-offs
Operational ReadinessSupport, monitoring, runbook
Resource ReadinessTeam availability, on-call rosters

Criterion statuses

ButtonStatusMeaning
✅GOCriterion is met
❌NO-GOCriterion is not met — blocks go-live
⏳PENDINGNot yet assessed

Overall decision logic

  • If any criterion is NO-GO → overall is NO-GO
  • Else if any criterion is PENDING → overall is PENDING
  • Else all are GO → overall is GO

Click Finalize Decision to lock the decision with a timestamp. The decision and timestamp appear in the Go/No-Go report.

Task Performance Report

Designed for post-cut-over review. For each task it shows:

  • Planned Start / Actual Start with start variance
  • Planned End / Actual End with end variance
  • Planned Duration / Actual Duration with duration variance
  • Summary: tasks started, completed, finished early, on time, late
  • Average planned duration vs average actual duration

Variances are colour-coded: green for early/on-time, red for late.

Printing and PDF export

Click 🖨️ Print Report in the report modal footer. In the browser print dialog, select Save as PDF as the destination to produce a PDF file.

Task categories

Categories group tasks and provide colour-coded badges in all views and reports.

  • Add — type a name, pick a colour, click + Add
  • Rename — click the name field; saved when you click away
  • Recolour — click the colour swatch; saved automatically
  • Delete — click 🗑️ (tasks referencing the old slug are unaffected)

Issue categories

Issue categories work identically to task categories but apply only to the Issues view and Issues report. They are configured separately so issue types can differ from task types.

Default issue categories: Technical Data Integration Security Operational Resource Other

17Data Import & Export

Export to JSON

Click 📤 Export JSON in the sidebar footer. Downloads a full snapshot of the active project. Filename: [project-name]-[YYYY-MM-DD].json.

Use this to back up a project, share it with another user, or archive a completed project.

Import from JSON

Click 📥 Import and select a previously exported .json file. The import:

  • Loads all data into the currently active project
  • Backfills missing task numbers sequentially
  • Re-snaps dependent task start dates to dependency end + 1 second
  • Auto-calculates missing durations from start/end dates
  • Updates the project name in the registry
Warning: Import overwrites the current project's data. Export a backup first if you need to keep it.

Export to CSV

Click 📊 Export CSV ▾ and choose a data type from the dropdown.

ResourcesName, Role, Email, Phone, Availability, Status, Notes, Assigned Tasks
RisksID, Description, Severity, Probability, Impact, Mitigation, Owner, Status, Created
IssuesID, Title, Category, Priority, Status, Owner, Raised Date, Due Date, Resolution, Description
DecisionsID, Title, Status, Decided By, Impact, Options, Outcome, Created Date, Decided Date, Description

Files are UTF-8 encoded with a BOM for correct display in Microsoft Excel. Filename: [project-name]-[type]-[YYYY-MM-DD].csv.

Task statuses are fixed and cannot be configured.

StatusMeaning
Not StartedWork has not begun

Date & time format

All dates and times are displayed in the project timezone in 24-hour format: DD/MM/YYYY HH:MM:SS

Duration format

Durations are entered and displayed as HH:MM:SS. Example: 02:30:00 = 2 hours 30 minutes.

Data storage

All data is stored in your browser's localStorage. It is not synced to a server or shared with other browsers or users. To transfer data between machines, use Export JSON and Import.

Warning: Clearing your browser's site data will permanently delete all projects. Export regularly to keep backups.
